# Development Guide

Developer conventions and implementation notes for the `evfuse` package.

## Key Conventions

- Parameter-major ordering throughout (parameter index varies fastest):
  index `(j-1)*L + i` for param j at site i
- Log-rho parameterization for optimization (ensures rho \> 0)
- Lambda = 300 km Wendland C4 (order-2) taper for bootstrap W
- B = 500 bootstrap replicates
- 20 multi-start random restarts from log-uniform\[50, 5000\] km
- Prediction grid: filtered coastal points at 15 km spacing, lon -98 to
  -66, lat 24 to 46
- Seeds: bootstrap (42), retry loop (101-105), multi-start (2026),
  return level simulation (123)
- `source_params` abstraction: named list mapping source labels to
  parameter indices (e.g., `list(NOAA = 1:3, ADCIRC = 4:6)`); all core
  functions accept this via `dat$source_params`
- Unicode labels in figures (μ₀, ξ, σ) require
  [`ragg::agg_png`](https://ragg.r-lib.org/reference/agg_png.html)
  device in ggsave

## Implementation Notes

- [`extRemes::fevd`](https://rdrr.io/pkg/extRemes/man/fevd.html) returns
  observed information matrix directly (positive definite), not Hessian
  of negative log-likelihood. Use `solve(fit$results$hessian)` directly
  for vcov.
- Delta method for log_sigma transformation: J = diag(1, 1/sigma, 1)
- Rho optimized in log space: `pack_params` stores `log(rho)`,
  `unpack_params` exponentiates. Gradient uses chain rule.
- Analytic gradient contracts Q = V_inv - alpha\*alpha^T with Omega_i
  into pxp H matrices. See Section 3.3 and Supporting Information
  Section S1 (Eqs S1-S4).
- Sigma_obs built directly in observed space (never form full Lp x Lp).
- nll/nll_grad share a cache keyed on parameter vector to avoid
  redundant Cholesky.
- All core functions generalized via `source_params`; defaults match
  NOAA/ADCIRC case.
- [`wendland2()`](https://briannathanwhite.github.io/evfuse/reference/wendland2.md)
  implements Wendland C4 (psi\_{2,2} from Furrer et al. 2006), not C2.

## Nonstationary Extensions

Two nonstationary Stage 1 variants are available:

- **Linear time trend** (`fit_gev_detrended`): fits μ(t) = μ₀ + μ₁(t −
  t_ref) at NOAA sites, discards μ₁ and passes (μ₀, log σ, ξ) to
  Stage 2. Used in the primary manuscript analysis.
- **Covariate sensitivity** (`fit_gev_ns`): fits μ(t) = μ₀ +
  μ₁·covariate(t), retains μ₁ in θ̂ (4 columns). Supports asymmetric
  `source_params` (e.g., `list(NOAA = 1:4, ADCIRC = 5:7)` for p=7 model
  with 42 optimization parameters). Used for the forecasting extension.

Key implementation details for the covariate pipeline: -
`bootstrap_W_ns`: Gamma uses union ordering (4 params/site); ADCIRC μ₁
positions are NA → [`cov()`](https://rdrr.io/r/stats/cor.html) produces
NA - `embed_W`: detects unobserved params via
[`is.na()`](https://rdrr.io/r/base/NA.html) on W_tap diagonal, maps only
observed entries - `taper_W` must use `p = 4` (not default 3) for the
nonstationary bootstrap - `compute_return_levels_ns`: 4-dim delta method
gradient via chain rule

## Data

- Source: `data(coast_data)` or `data-raw/combined_max.csv`
- 129 sites: 29 NOAA, 100 ADCIRC
- 34-43 years of annual maxima per site

## Development

- Iterate with `devtools::load_all()` and `devtools::test()`
- Run tests after every change (102 tests in
  `tests/testthat/test-core.R`)
- R CMD check should pass with 0 errors, 0 notes (2 warnings from
  vignette inst/doc when pandoc is unavailable are expected)
